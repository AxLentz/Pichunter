# 规则: Agent 行为准则 (Agent Behavior)

## 1. 协作准则 (Collaboration)

- **透明化 (Transparency)**: 关键操作（如修改配置、删除文件、执行 Git 提交等）前，必须简述原因并获得授权。
- **授权优先 (Authorization First)**: 对于具有不可逆风险的操作（如覆盖整个文件内容 \`Overwrite\` 或执行 \`Git Commit\`），即使之前得到过模糊授权，在执行前也必须展示最终变更摘要，获得“二次确认”后方可执行。
- **规则维护协议 (Rules Maintenance)**: 在修改或新增任何规则文件（Rules）前，必须首先阅读并严格遵守 \`00-A-RulesGuideline.md\` 中的格式、命名与视觉规范（如 YAML 元数据、50 个短横线分隔线等）。
- **最小干预 (Minimal Intervention)**: 在执行修复或功能开发时，严禁顺带进行无关的代码重构、注释美化。除非用户明确要求，禁止对非目标区域进行任何改动。
- **工具优先 (Tooling First)**: 严禁手动调整导包顺序、缩进等风格问题。所有格式化工作必须交给项目已配置的工具（如 Ruff, Pre-commit）完成，禁止在 Git Diff 中产生格式噪音。
- **分步执行 (Step-by-Step)**: 复杂任务应拆分为可验证的小步，每步完成后提供反馈。

--------------------------------------------------

## 2. 核心安全禁令 (Safety First)

1. **API Key 安全**: 严禁将 \`.env\` 文件或任何明文 API Key 提交至 Git 仓库。
2. **操作溯源**: 所有命令必须在项目根目录或指定的后端/前端目录下执行，严禁在系统敏感目录操作。
3. **静默执行禁止**: 严禁在未告知用户的情况下修改 \`.gitignore\` 或 \`.github/workflows\` 等关键配置文件。

--------------------------------------------------

*本准则作为 Agent 的底层行为底座，始终处于激活状态。*
