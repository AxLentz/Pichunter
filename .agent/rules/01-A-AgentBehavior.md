# 规则: Agent 行为协议 (Agent Behavior)

## 1. 核心安全禁令 (Safety First)
- **严禁静默批量修改**: 禁止使用复杂的 sed, awk 或正则表达式对项目文件进行自动化的批量重写。
- **严禁未经确认的删除**: 在执行 rm 或清空文件内容的命令前，必须明确告知用户。
- **写入即检查**: 任何文件修改后，必须通过 view_file 或相关工具检查内容的完整性。

--------------------------------------------------

## 2. 协作准则 (Transparency)
- **授权优先**: 在涉及修改核心逻辑、安装新依赖或创建关键配置文件等重大“侵入性”的操作前，必须先向用户阐明方案并征得明确同意。对于**颗粒度较小**的操作（如修正导入顺序、添加日志埋点、或在已讨论方案范围内的细节修正），可以在确保安全的前提下直接执行以提高效率。
- **原子执行**: 避免将多个复杂的文件操作合并在一条指令中，优先使用分步指令以降低失败率。
- **验证闭环**: 编码完成后，主动提供具体的验证操作或测试脚本给用户。

--------------------------------------------------

*注：本守则旨在确保 AI Agent 在开发过程中的行为高度安全且透明。*
